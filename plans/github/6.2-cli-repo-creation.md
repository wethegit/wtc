# 6.2 CLI Repo Creation

## Goal

Add the CLI equivalent of Phase 6.1 repo creation.

This phase mirrors only the basic TUI workflow: create a repo from an approved owner template. It does not include branch protection, Teamwork linking, or local clone.

## Scope

- Add a `repo` CLI command group.
- Add `wtc repo create`.
- Reuse `github.repoOwner` from user config.
- Reuse API functions from `src/api/github/repos.ts`.
- Keep the command scriptable first.

## Proposed Command

```bash
wtc repo create <name> --template <template-name> --visibility private --description "Optional description"
```

Flags:

- `--template <template-name>`: template repo name under `github.repoOwner`.
- `--visibility <private|public>`: defaults to `private`.
- `--description <text>`: optional.
- `--json`: print machine-readable output.
- `--open`: optional browser open after creation, if useful.

Behavior:

- Loads user config.
- Requires `github.repoOwner`.
- Verifies the requested template exists and has `is_template === true`.
- Creates the repo under `github.repoOwner`.
- Prints the created repo URL.

## Deferred

- Interactive TTY prompts.
- Branch protection flags.
- Teamwork/project linking flags.
- Clone flags.

## Open Questions

- Should `--template` accept only a repo name (`starter`) or also `owner/repo`?
- Should `--open` be included immediately or deferred?
- Should missing flags fail fast or prompt when stdout is a TTY?
