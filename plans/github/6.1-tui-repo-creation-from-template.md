# 6.1 TUI Repo Creation From Template

## Goal

Create a new GitHub repository from an approved organization template, or as a truly empty repository, through the TUI.

This phase is intentionally narrow: TUI first, no CLI, no local clone, no branch protection, and no Teamwork/project linking.

## Decisions

- Repo creation is TUI-only in this phase.
- The approved owner is configured in user config as `github.repoOwner`.
- New user config defaults `github.repoOwner` to an empty string.
- Settings should show `wethegit` as the placeholder/example.
- The configured `github.repoOwner` is both the template source and the new repo target owner.
- Only repos with `is_template === true` are selectable.
- Template selection includes a `none` option for creating a truly empty repo.
- Visibility is user-selectable: `private` or `public`, with `private` as the default.
- Description is optional and skippable.
- Created repo opens through a success dialog with an `Open` action.
- If GitHub auth or `github.repoOwner` is missing, the GitHub page points the user to Settings.
- Template creation copies only the default branch: `include_all_branches: false`.

## Config Scope

Add a user-level GitHub config section:

```yaml
github:
  repoOwner: ""
```

Schema shape:

```ts
github: {
  repoOwner: string;
}
```

Settings TUI field:

- Field: `github.repoOwner`
- Placeholder: `wethegit`
- Description: `Approved GitHub org used for repo templates and new repos.`

## API Scope

Add `src/api/github/repos.ts`.

### `getGitHubTemplateRepos(owner)`

Reads template repos from the configured owner.

Behavior:

- Uses Octokit through `getOctokit()`.
- Calls `octokit.rest.repos.listForOrg` for the configured owner.
- Filters to `repo.is_template === true`.
- Does not cache initially.
- Throws normal Octokit errors so UI can show clear failure messages.

Return shape:

```ts
interface GitHubTemplateRepo {
  owner: string;
  name: string;
  fullName: string;
  description: string | null;
  htmlUrl: string;
  private: boolean;
}
```

### `createGitHubRepoFromTemplate(input)`

Creates a new repo under the configured owner from a selected template.

Input shape:

```ts
interface CreateGitHubRepoFromTemplateInput {
  templateOwner: string;
  templateRepo: string;
  owner: string;
  name: string;
  description?: string;
  private: boolean;
}
```

### `createGitHubRepo(input)`

Creates a truly empty repo under the configured owner.

Input shape:

```ts
interface CreateGitHubRepoInput {
  owner: string;
  name: string;
  description?: string;
  private: boolean;
}
```

Behavior:

- Uses Octokit through `getOctokit()`.
- Calls `octokit.rest.repos.createInOrg`.
- Sends `auto_init: false` so GitHub does not create a README or initial commit.
- Lets GitHub enforce exact repo naming and permission constraints.

Behavior:

- Uses Octokit through `getOctokit()`.
- Calls `octokit.rest.repos.createUsingTemplate`.
- Sends `include_all_branches: false`.
- Lets GitHub enforce exact repo naming and permission constraints.

Return shape:

```ts
interface CreatedGitHubRepo {
  name: string;
  fullName: string;
  htmlUrl: string;
  cloneUrl: string;
  sshUrl: string;
}
```

## TUI Scope

Replace the placeholder GitHub page with a focused creation screen.

Page behavior:

- Shows GitHub auth status.
- Shows configured `github.repoOwner` when available.
- If GitHub auth is missing, show a message directing the user to Settings.
- If `github.repoOwner` is missing, show a message directing the user to Settings.
- When ready, expose a `Create repo from template` action.

Workflow:

1. Load templates from `github.repoOwner`.
2. Select a template with `DialogSelect`, or select `none` for a truly empty repo.
3. Enter repo name with `DialogInput`.
4. Enter optional description or skip it.
5. Select visibility with `DialogSelect` (`private` first).
6. Confirm creation with `ConfirmDialog`.
7. Show `LoadingDialog` while creating.
8. Show success dialog with repo URL and `Open` button.
9. `Open` opens the created repo URL in the browser.

Command palette:

- Keep navigation command to the GitHub page.
- Do not launch creation directly from the command palette in this phase unless it falls out naturally.

## Out Of Scope

- CLI commands.
- Branch protection.
- PR/review/status check rules.
- Teamwork/project linking.
- Local clone.
- Manual template entry.
- Personal account repo creation.
- GitHub Enterprise `internal` visibility.
- Template/repo list caching.

## Definition Of Done

- User can configure `github.repoOwner` in Settings.
- GitHub page clearly reports missing auth or missing repo owner.
- GitHub page lists template repos from `github.repoOwner`.
- User can create a private or public repo from a selected template.
- User can create a private or public blank repo by selecting `none`.
- Description can be skipped.
- Created repo success dialog includes an `Open` action.
- `bun run check` passes.
- `bun run lint` passes.
- No tests are added.
