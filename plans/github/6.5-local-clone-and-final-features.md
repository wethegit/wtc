# 6.5 Local Clone And Final Features

## Goal

Add local clone and final repo creation workflow conveniences after basic creation, repo rules, CLI parity, and project linking are stable.

## Scope Candidates

- Prompt to clone after repo creation.
- Choose clone destination directory.
- Choose SSH or HTTPS clone URL.
- Open the local directory after cloning if useful.
- Surface clone progress and failures clearly.
- Add any final workflow shortcuts discovered in previous phases.

## TUI Flow

Potential flow after repo creation/linking:

1. Ask whether to clone locally.
2. Select SSH or HTTPS.
3. Enter local destination path.
4. Run `git clone`.
5. Show success or failure.

## CLI Scope

Potential flags once behavior is proven:

- `--clone`
- `--clone-dir <path>`
- `--clone-protocol <ssh|https>`

## Open Questions

- Should clone default to SSH or HTTPS?
- What default parent directory should be used?
- Should WTC create the parent directory if missing?
- Should clone be available in TUI first, CLI first, or both together?
