# 5.5 Branch and PR Workflow

## Goal

Create branches and draft pull requests from selected Teamwork tasks while encouraging correct time tracking and useful PR context.

## Branch From Task

From a selected Teamwork task:

- Ask whether to start a timer for the task.
- If another timer is running, ask whether to pause it before starting the task timer.
- Ask which source branch to branch from.
- Create a branch name from the task ID and title.
- Store the task-to-branch association so the TUI can show branch context beside the task later.

## Draft PR Flow

After branch creation:

- Ask whether to create a draft PR immediately.
- Ask which target branch the PR should merge into.
- Create the draft PR.
- Include the Teamwork task link in the PR body.
- Include the code review/time tracking task link in the PR body when WTC can identify it.
- Include a reminder that time tracking is required.

## Data Storage

Project config can act as the shared project database when the mapping should be committed and visible to the team.

Possible future shape:

```yaml
teamwork:
  projectId: 362632
  pinnedTaskLists:
    - name: General Tasks
      id: 1597639
  taskBranches:
    - taskId: 123456
      branch: feature/TASK-123456-example
      pullRequestUrl: https://github.com/org/repo/pull/123
```

Do not add this config shape until branch/PR behavior is actually implemented.

## GitHub Integration

- Reuse future GitHub provider/client behavior rather than mixing GitHub HTTP details into Teamwork modules.
- PR creation needs source branch, target branch, title, body, and draft status.
- Branch creation needs current repo context and source branch choice.

## CLI Scope

Add branch and PR CLI commands after the TUI dialog workflow proves the required prompts, defaults, and timer interactions.

```bash
wtc teamwork task branch <taskId|taskUrl>
wtc teamwork task branch <taskId|taskUrl> --start-timer
wtc teamwork task branch <taskId|taskUrl> --start-timer --switch-timer
wtc teamwork task pr <taskId|taskUrl> --draft
```

- Keep branch and PR commands under `task` because the workflow starts from a Teamwork task.
- `--start-timer` asks the workflow to start tracking time for the task.
- `--switch-timer` allows the command to pause an existing running timer before starting the task timer.
- Branch source and PR target can use prompts in TTY mode and explicit flags later for scripting.
