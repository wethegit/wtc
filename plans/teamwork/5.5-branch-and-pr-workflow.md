# 5.5 Branch and PR Workflow

## Goal

Create branches and draft pull requests from selected Teamwork tasks while encouraging correct time tracking and useful PR context.

## Implementation Order (Sub-Phases)

### 5.5.1 — GitHub API Foundation

**Scope:** Auth, client, and config CLI for the GitHub API, following the Teamwork foundation pattern (5.1).

**Deliverables:**

- Install `@octokit/rest`
- `src/api/github/auth.ts` — `Bun.secrets` with `service: "wtc"`, `name: "github-api-token"`, mirroring `src/api/teamwork/auth.ts` exactly (get/set/delete/status)
- `src/api/github/client.ts` — wraps Octokit instantiation, exports `getOctokit(): Octokit`
- Add `"github"` to `CONFIG_AUTH_PROVIDERS` in `src/cli/commands/config.ts`
- Support `wtc config auth set github`, `wtc config auth status github`, `wtc config auth delete github`
- Token docs added to `USAGE.md` already

### 5.5.2 — Git Utilities & Task-Branch Cache

**Scope:** Local git operations and persistent task-to-branch mapping.

**Deliverables:**

- `src/utils/git.ts` — `detectRepo(projectDir): string | null`, `currentBranch(): string`, `createBranch(name: string)`, `pushBranch(name: string)`, `getDefaultBranch(owner, repo): string`
- GitHub handle detection via `GET /user`, cached in `github-user.json` (same pattern as Teamwork user cache)
- `src/api/github/task-branches.ts` — cache file `task-branches.json` in the shared cache dir, keyed by repo URL:
  ```ts
  { [repoUrl: string]: { [taskId: number]: { branch: string, prUrl?: string, createdAt: number } } }
  ```
- Functions: `getTaskBranch(repoUrl, taskId)`, `setTaskBranch(repoUrl, taskId, branch)`, `setTaskBranchPrUrl(repoUrl, taskId, prUrl)`

### 5.5.3 — Branch from Task (TUI Workflow)

**Scope:** Interactive TUI dialog to create a branch from a selected Teamwork task.

**Deliverables:**

- New `DialogInput` component for text input (branch name), following `<input>` pattern from `DialogSelect`
- Branch action wired to `ProjectTab` and `MyWorkTab` via `ctrl+b`
- Dialog flow:
  1. **Timer prompt** — "Start timer for this task?" (reuse `ConfirmDialog` + `toggleTimer`)
  2. **Branch name dialog** — prefilled `{handle}/tw{taskID}`, editable. Footer note: "Custom branch names aren't shared with other WTC users — only standard `{handle}/tw{id}` names are."
  3. **Create branch** — `git checkout -b <name>` from current HEAD, `git push -u origin <name>`
  4. **Cache write** — `setTaskBranch(repoUrl, taskId, branchName)`
  5. **Post-branch prompt** — "Create draft PR now?" → if yes, triggers 5.5.4 flow
- Source branch: always the current HEAD for v1

### 5.5.4 — Draft PR (TUI Workflow)

**Scope:** Interactive TUI dialog to create a draft PR from a mapped task.

**Deliverables:**

- PR creation via Octokit: `octokit.rest.pulls.create({ owner, repo, title, head, base, body, draft: true })`
- PR dialog:
  - Target branch selection — default to repo default branch, option to pick from recent branches
  - PR body construction:

    ```
    Closes #{taskUrl}

    ## Time Tracking
    Remember to log time for this task in Teamwork.
    ```

- Wire action to tasks that have a branch mapping

### 5.5.5 — CLI Commands

**Scope:** CLI equivalents of the TUI workflows.

**Deliverables:**

- `wtc teamwork task branch <taskId|taskUrl>` — flags: `--start-timer`, `--switch-timer`, `--name`
- `wtc teamwork task pr <taskId|taskUrl>` — flags: `--draft`, `--target`
- Prompt for branch name / target branch in TTY mode; skip prompts when flags are provided

## Key Design Decisions

- **Task-to-branch storage:** Local cache only (`task-branches.json`), not committed to `.wtc.yaml`. Mappings are per-user. Custom branch names (outside `{handle}/tw{id}`) are not discoverable by other WTC users.
- **Branch naming convention:** `{userGithubHandle}/tw{taskID}` by default, editable in dialog.
- **Source branch:** Always the current git HEAD for v1.
- **GitHub handle:** Fetched once from `GET /user` and cached locally.
- **Timer integration:** Reuse `useTaskTimer` from the shared hooks. The branch dialog prompts to start a timer for the task before branching.

## Open Questions

- Should draft PR creation happen immediately after branch creation by default, or always ask? (Plan: always ask via post-branch prompt.)
