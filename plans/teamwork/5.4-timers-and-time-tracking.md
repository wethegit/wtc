# 5.4 Timers and Time Tracking ✅ Complete

## Goal

Make time tracking fast from the TUI, especially for recurring project tasks and assigned tasks. Timers are managed with Teamwork's native timer API so timer state is shared with Teamwork directly.

## Architecture Decision: Teamwork Native Timers

- **Teamwork timer API** (`src/api/teamwork/timers/api.ts`): list, start, pause, resume, complete, and delete timers.
- **No local timer cache**: `teamwork-local-timers.json` and `src/api/teamwork/timers/local.ts` were removed.
- **Local elapsed display**: timer duration is calculated locally from Teamwork's `duration` and `lastStartedAt` fields to avoid polling the API every second.

## Start Timer From Task

- `ctrl+t` on a selected task toggles that task's Teamwork timer.
- If no timer exists for the task, WTC creates one and starts it.
- If a stopped timer exists for the task, WTC resumes it.
- If the task's timer is already running, WTC pauses it.
- If another timer is running, Teamwork pauses it server-side when the selected task's timer starts or resumes.
- Task metadata shows a compact `⏱` badge next to tasks with timers.

## Timer Tab

A dedicated Teamwork timers tab shows native Teamwork timer entries:

- Running timer listed first (with flashing `⏱` indicator)
- Stopped timers below, sorted by last start time (newest first)
- `↑`/`↓` selects timers.
- `ctrl+t` toggles the selected timer between running and paused.
- `ctrl+s` completes/submits the selected timer to Teamwork as a timelog entry.
- `ctrl+d` deletes the selected Teamwork timer after confirmation.
- `ctrl+o` opens the Teamwork timesheet in the browser.

## Visual Indicators

- Running timer: flashing `⏱`
- Stopped timer: dim `⏱`
- No timer: nothing shown

## CLI Scope

Timer CLI commands mirror the native Teamwork behavior and use task references as the primary user-facing handle:

```bash
wtc teamwork timer list
wtc teamwork timer start <taskId|taskUrl>
wtc teamwork timer stop <taskId|taskUrl>
wtc teamwork timer submit <taskId|taskUrl>
wtc teamwork timer delete <taskId|taskUrl>
wtc teamwork timesheet open
```

- `timer start` resumes an existing timer for the task or creates one if needed.
- `timer stop` pauses the running timer for the task.
- `timer submit` completes the timer and creates the Teamwork timelog entry.
- `timer delete` deletes the Teamwork timer without submitting it.
- If multiple timers match the same task, WTC prefers the running timer, then the most recently started timer.

## Teamwork API Endpoints

- `GET /projects/api/v3/me/timers.json` — list timers for the authenticated user.
- `POST /projects/api/v3/me/timers.json` — create/start a timer.
- `PUT /projects/api/v3/me/timers/{timerId}/pause.json` — pause a timer.
- `PUT /projects/api/v3/me/timers/{timerId}/resume.json` — resume a timer.
- `PUT /projects/api/v3/me/timers/{timerId}/complete.json` — complete a timer and create the timelog entry.
- `DELETE /projects/api/v3/me/timers/{timerId}.json` — delete a timer.
