# 5.1 Teamwork Foundation

Status: Complete.

## Goal

Make WTC aware of the current Teamwork project and user auth state without adding task or timer mutations yet.

## Current Scope

- ✅ Store Teamwork API tokens in Bun OS secrets, never YAML.
- ✅ Keep the Teamwork site fixed to We The Collective's Teamwork instance.
- ✅ Add `teamwork.projectId` to project config.
- ✅ Add a Teamwork route with `my-work` and `project` tabs.
- ✅ Restore Teamwork route tab state per project directory.
- ✅ Fetch and cache Teamwork project metadata by project ID.
- ✅ Show project config status, auth status, project links, and project metadata in the Project tab.

## API Shape

- Use `getTeamworkProjectMetadata(projectId)` for project metadata reads.
- The `get*` function owns cache lookup, network fetch, cache save, stale-cache fallback, and return shape.
- Shared Teamwork HTTP behavior belongs in `src/api/teamwork/client.ts`.

## Cache

- Project metadata uses a small endpoint-specific cache under the shared WTC cache directory.
- `wtc cache clean` should remove this cache with the rest of runtime cache data.
- Do not introduce a generic Teamwork cache layer until multiple Teamwork endpoints need shared invalidation behavior.

## Open Questions

- ✅ Confirm the exact Teamwork project metadata response shape enough for the current display.
- Manual refresh is deferred until real usage shows the automatic load is insufficient.
